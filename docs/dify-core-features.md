# Dify 核心功能模块梳理

> 整理时间: 2026-04-09
> 信息来源: GitHub 官方文档、技术架构分析、官方文档站点

## 一、平台定位

Dify 是由 LangGenius 开发的开源 LLM 应用开发平台，融合了 **BaaS（Backend-as-a-Service）** 和 **LLMOps** 理念，帮助企业与开发者快速构建、部署和管理生产级的生成式 AI 应用。

- **GitHub**: https://github.com/langgenius/dify
- **官网**: https://dify.ai
- **文档**: https://docs.dify.ai

---

## 二、应用层（5种应用类型）

| 模块 | 说明 |
|------|------|
| **聊天助手 (Chatbot)** | 基于大模型的对话式 AI，支持多轮对话和知识库增强，适用于客服、问答场景。 |
| **文本生成 (Text Generator)** | 专注单轮内容创作，支持模板化生成文案、报告、邮件等。 |
| **智能代理 (Agent)** | 具备工具调用能力，基于 CoT/ReAct 策略自主规划任务，可调用外部工具完成复杂目标。 |
| **对话流 (Chatflow)** | 面向多轮对话场景的工作流编排，支持记忆能力，适合客服、交互式助手。 |
| **工作流 (Workflow)** | 面向单轮自动化任务的工作流，适合数据处理、批处理流水线。 |

---

## 三、编排引擎层（工作流节点）

| 模块 | 说明 |
|------|------|
| **LLM 节点** | 调用大模型生成回答，支持温度、Top P、最大 token 等参数精细控制。 |
| **Prompt IDE** | 可视化 Prompt 编辑器，支持 Jinja2 模板变量和动态逻辑，内置多模型测试沙箱。 |
| **知识检索节点** | 从向量知识库中检索相关内容，供下游节点使用。 |
| **问题分类器** | 意图识别节点，根据用户输入自动路由到不同分支。 |
| **代码节点** | 在沙箱环境中执行 Python/Node.js 脚本，实现自定义数据处理逻辑。 |
| **条件/分支节点 (IF/ELSE)** | 基于布尔表达式控制流程走向，实现复杂的业务逻辑判断。 |
| **工具节点** | 调用外部 API、搜索引擎、数据库等 50+ 预置工具或自定义扩展。 |
| **HTTP 请求节点** | 直接调用外部 REST API，无需编写代码。 |
| **模板节点** | 文本格式化转换，支持变量替换和字符串处理。 |
| **人机协同节点 (HITL)** | v1.13.0+ 新增，支持流程暂停等待人工审批后继续执行。 |

## 四、AI 能力层

| 模块 | 说明 |
|------|------|
| **模型网关 (Model Gateway)** | 统一接入层，支持 OpenAI、Claude、DeepSeek、通义千问、Llama 等数百种模型，一键切换。 |
| **RAG 引擎** | 端到端检索增强生成管道，支持 20+ 文档格式（PDF、PPT、Word、Excel、Markdown 等）。 |
| **知识库管理** | 支持文件、Notion、网页等多种数据源导入，提供通用分块、父子分层分块、Q&A 结构化提取等策略。 |
| **Agent 框架** | 基于 LLM Function Calling 或 ReAct 范式定义 Agent，提供 50+ 内置工具（Google 搜索、DALL·E、Stable Diffusion、WolframAlpha 等）。 |

---

## 五、运维与治理层

| 模块 | 说明 |
|------|------|
| **LLMOps** | 实时监控应用日志、性能指标和成本用量，支持基于生产数据的持续优化和 Prompt 版本管理。 |
| **A/B 测试** | 对比不同提示词或模型版本的表现，数据驱动优化。 |
| **标注与优化** | 基于用户反馈和人工标注持续改进提示词、数据集和模型。 |
| **BaaS 后端即服务** | 所有功能提供 REST API，可轻松集成到自有业务系统中。 |

---

## 六、企业级功能

| 模块 | 说明 |
|------|------|
| **SSO/访问控制** | OAuth2.0 + JWT 认证，RBAC 三级权限模型。 |
| **数据安全** | FPE 字段级加密，数据隔离存储，满足合规要求。 |
| **审计日志** | Elasticsearch 分布式存储，满足等保三级 6 个月留存要求。 |
| **多租户支持** | 团队协作开发，权限精细化管理。 |
| **用量配额控制** | 基于租户的资源限制与计费管理。 |
| **SSRF 代理网络** | 隔离内部网络，防止未授权外访，保障沙箱执行安全。 |

---

## 七、基础设施层

| 模块 | 说明 |
|------|------|
| **Celery 执行引擎** | v1.13.0+ 工作流在分布式 Worker 中运行，非 API 进程。 |
| **Redis Pub/Sub** | 流式结果通过 Redis 传回，支持实时输出。 |
| **PostgreSQL** | 工作流状态图定义和持久化存储。 |
| **安全沙箱** | 代码节点在隔离环境中执行，保障平台安全。 |

## 十、部署方式

1. **Dify Cloud（云服务）**：零配置即可使用，沙盒计划包含 200 次免费 GPT-4 调用
2. **社区版自托管**：Docker Compose 一键部署，最低配置 2核 CPU / 4GB 内存
3. **企业版**：提供 SSO、访问控制等企业级功能，支持私有化部署
