审查代码并提供反馈。

目标: $ARGUMENTS (若未指定则审查当前 git diff)

## 审查重点

1. **Bug 与错误**: 逻辑错误、空值检查、边界情况
2. **安全性**: 输入验证、注入风险、敏感数据暴露
3. **性能**: 明显的低效问题、N+1 查询、内存泄漏
4. **可读性**: 命名规范、复杂度、文档需求

## 步骤

1. 若提供了文件路径，读取该文件
2. 若未提供路径，运行 `git diff` 查看当前变更
3. 根据审查重点分析代码
4. 提供结构化反馈

## 输出格式

```markdown
## 代码审查

### 概述
[一句话总体评价]

### 发现的问题

#### 严重（必须修复）
- [问题]: [位置] - [简要说明]

#### 警告（建议修复）
- [问题]: [位置] - [简要说明]

#### 建议（可选优化）
- [建议]: [位置] - [简要说明]

### 优点
- [正面观察]
```

## 指南
- 明确指出位置（尽可能精确到 file:line）
- 提供可操作的反馈，而非单纯批评
- 除非影响可读性，否则不要纠结于代码风格
- 认可看到的优秀实践
